## Summary

Rebuilt Apex Scheduler as a self-serve, multi-tenant SaaS platform. Each business is isolated via Supabase RLS scoped to `business_id`. Owners onboard on first login, invite staff by code, configure roles, and manage locations. Free/Pro tier gating is enforced in the app layer with an upgrade UI shell.

## Manual QA (requires live Supabase)

2. Sign up as Owner A → create business → confirm dashboard shows business name.
3. Sign up as Owner B → create second business → confirm zero shared shifts/staff.
4. Owner A creates invite → Staff signs up with code → staff sees only Owner A schedule.
5. Free tier: add 10 staff (or mock count) → invite blocked with upgrade prompt.
6. Free tier: attempt second location → blocked with upgrade prompt.
7. Beta upgrade button sets `plan_tier = 'pro'` on businesses row.
